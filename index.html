Lesson 1 - Introduction to HTML

How the Web Works?
The web is made up of:
  • HTML which "glues" everything together
  • URLs
  • HTTP
  • Web Applications

Hyperlinks serve as the glue holding together all pages of the web. Many types of files can be found on the web, including Plain Text, HTML, Images, Videos, and Music.

There are 3 main pieces of Web- Your Computer , Internet and Servers

Your computer running a web browser (Internet Explorer or Google Chrome) and browser makes a request to to servers over internet which is the largeest computer network using HTTP protocol. Servers send the response to your computer through HTTP. HTTP (HyperText Transfer Protocol) is the protocol that helps in communicating between your computer and servers. The servers are also like computers but they are used for hosting the files that make up the web .

HTML

HTML stands for Hyper Text Markup Language. It consists of Markup tags,Web Contents,Images and Videos and References to other documents called Hyperlinks.HTML documents are made up by HTML elements.HTML elements are written with start tag, end tag and the content between the start tag and end tag.

Here some of the important concepts of HTML:
  1. Tags: Tags modify content. Here are some examples:
    • Bold <b> + content + </b>
    • Italics <em> + content + </em>
    • Links <a href="value"> + content + </a>. Note that this uses "href."
    • Images <img src="url" alt="text">+ content. Note that this uses "src."
    • Break <br> which ends lines
    • Paragraph <p> which makes text blocks
  2. Element: Everything within a set of tags.
  3. Attribute: Property of an HTML Element.
  4. White Space: Text that takes up multiple lines while typing code will show as one line on a webpage unless broken up with tags.

Inline Vs Block Elements

An Inline element does not start on a new line and only takes up as much width as necessary.

Examples of inline elements:
  • <br>
  • <span>
  • <img>
A Block-level element always starts on a new line and takes up the full width available (stretches out to the left and right as far as it can. Examples of block elements:
  • <div>
  • <h1-h6>
  • <form>
  • <p>

The <div> element is a block-level element that is often used as a container for other HTML elements. The <div> element has no required attributes, but style and class are common. When used together with CSS, the <div> element can be used to style blocks of content:

The <span> element is an inline element that is often used as a container for some text. The <span> element has no required attributes, but style and class are common. When used together with CSS, the <span> element can be used to style parts of the text

Lesson 2- Creating a Structured Document with HTML

Developer Tools (in the Browser)

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them

The "tree-like structure" of HTML

The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. You can draw this relationship like a family tree. My mother had multiple children. So did her mother, and so on... In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

How broswers classify HTML. This is what developers talk about when they refer to "DOM," the Document Object Model.

The relationship between indented HTML and boxes

When you read an HTML document as text, you see a wave of changing indentations going up and down the left side of the document. The more indented an element is, the more deeply nested its corresponding "box" is.

Text Editors

Lesson 3-Adding CSS style to HTML structure

Avoiding Repetition

Avoiding repetition is important for a variety of reasons.

Avoiding Errors: When programmers don't avoid repetition, they will often have to do the same thing over and over. This may mean copy and pasting certain style attributes into many HTML elements or rewriting the same code (sometimes with minor differences) many times. This can lead to errors when the programmer decides to make a change to something. If they don't dilligently make that same change everywhere the repeated code appears, problems will arise.

Consistency: In this web page, I wanted every lesson to look the same. Instead of manually giving each one the same style, I was able to assign each lesson div the same class and then specify the style for that class in only one place! By avoiding repetion, it's now very easy for me to make changes to this style.

CSS
Cascading Style Sheet

CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements.
This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.

First, prep the document to make sure it will respond to the following changes:
  • Add DOCTYPE html to the top of the file
  • Add <html> directly below this tag
  • Add </html> below the final tag in the file Indent the HTML structure within HTML tags
  • Add <head> </head> below <html>
  • Add <link rel="stylesheet" type="text/css" href=""> and add the name of the CSS file after href
  • Add <body> </body> tag and move HTML structure within <body> tags
Then, you will write two files:
  • main.html
  • main.css
Box Model
The internet is made up of boxes. Tags produce four unique boxes:
  • Content: image or text
  • Padding: clears the area around the content. The space inside of the box that protects the content directly
  • Border: goes around the above to boxes.Color proprerties inherited from the box properties.
  • Margin: which is not affected by any background colors. "Space between boxes.
  • The Border + Padding + Content widths = the size of the element.
Code,Test & Refine
Coding has a good amount of trial and error built into the process. Considering this, here's a guide on how to code efficiently.
  • Look for natural boxes
  • Look for repeated style and semantic elements
  • Write your HTML
  • Apply styles (from biggest to smallest)
  • Fix things
  • Repeat the above three steps until the website looks like the design mock-up
